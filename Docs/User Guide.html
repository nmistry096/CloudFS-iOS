Getting the SDK

You can get the SDK by building the project from source.

  Build the project from source

You can download the project from our repository and build it. To checkout the latest source code:

git clone https://github.com/bitcasa/CloudFS-iOS.git

Using the SDK

If you don't have a CloudFS API account, register to gain access.

  Creating and linking the CloudFS Sessions

Sessions represent connections to CloudFS. They use a set of credentials that consists of an end point URL, a client ID and a client secret. These credentials can be obtained via the Bitcasa admin console.

session = [[CFSSession alloc] initWithServerURL:serverUrl clientId:appId clientSecret:appSecret];

A user can be linked to the session by authenticating using a username and a password.

[session authenticateWithUsername:email andPassword:password completion:^(NSString token, BOOL success, CFSError error){ //YOUR CODE }];

You can assert whether a user is linked to the session.

[session isLinkedWithCompletion:^(BOOL response, CFSError *error) { //YOUR CODE }];

The currently linked user can be unlinked from the session.

[session unlink];

Admin Operations

  Create Account

You need to set the admin credentials in order to perform admin operations. You can create end users for an admin/paid account. If ‘logInToCreatedUser’ is true, logs in to the user after creating it.

[session setAdminCredentialsWithAdminClientId:clientId adminClientSecret:adminSecret];

[session createAccountWithUsername:[self getRandomEmail] password:@"test123" email:nil firstName:nil lastName:nil logInTocreatedUser:NO WithCompletion:^(CFSUser user, CFSError error) { //YOUR CODE }];

  Update Account

You can update user information including user plan information.

[session setAdminCredentialsWithAdminClientId:clientId adminClientSecret:adminSecret];

[session updateUserWithId:ses.account.accountId userName:nil firstName:@"TestUserNameChanged" lastName:@"TestUserLastNameChanged" planCode:plan.planId WithCompletion:^(CFSUser *updatedUser, CFSError *error) { //YOUR CODE }];

Create plan

You can create custom user plans. These plans can be assigned to a user.

[session setAdminCredentialsWithAdminClientId:clientId adminClientSecret:adminSecret];

[session createPlanWithName:[self getRandomEmail] limit:@"1024" completion:^(CFSPlan *plan, CFSError *error) { //YOUR CODE }];

List plans

You can list all the custom plans created.

[session setAdminCredentialsWithAdminClientId:clientId adminClientSecret:adminSecret];

[session listPlansWithCompletion:^(NSArray *plans, CFSError *error) { //YOUR CODE }];

File System Operations

You need to create a session in order to perform file system operations.

  Get Root Folder

CFSFilesystem *fileSystem = [[CFSFilesystem alloc] initWithRestAdapter:restAdapter];

[fileSystem rootWithCompletion:^(CFSFolder root, CFSError error) { //YOUR CODE }];

  Get Specific Item

[fileSystem getItem:path completion:^(CFSItem item, CFSError error) { //YOUR CODE }];

  List Trash Items

You can list down the contents of Trash folder. Below example shows how to retrieve contents of the trash folder.

[fileSystem listTrashWithCompletion:^(NSArray items, CFSError error) { //YOUR CODE }];

  Get Shares

You can list down available Shares. Below example shows how to retrieve the list of shares.

[fileSystem listSharesWithCompletion:^(NSArray items, CFSError error) {//YOUR CODE}];

  Create Share

You can create a share by providing the path as shown in below example. A passworded share cannot be used for anything if the password is not provided. It doesn t make sense to create a share unless the developer has the password.

[fileSystem createShare:file.path password:SHARE_PASSWORD completion:^(CFSShare share, CFSError error) { //YOUR CODE }];

  Get Specific Share

You can get a share by providing the share key and the password (If available). A passworded share cannot be used for anything if the password is not provided.

[fileSystem retrieveShare:share.shareKey password:SHARE_PASSWORD completion:^(CFSShare share, CFSError error) { //YOUR CODE }];

Folder Operations

You need to create a session in order to perform folder operations.

  List Folder Contents

You can list the contents of a folder. This will return a list of top level folders and items in the specified folder.

[folder listWithCompletion:^(NSArray items, CFSError error) { //YOUR CODE }];

  Change Folder Attributes

You can change the attributes of a Folder by providing a hash map of field names and values. An example is given below.

[folder changeAttributes:newApplicationData ifConflict:VersionExistsIgnore];

  Copy Folder

You can copy a folder to a new location in the file system. If the destination conflicts with the copying folder you can either RENAME, OVERWRITE or FAIL the operation.

[folder copyToDestinationContainer:destinationFolder whenExists:CFSExitsRename completion:^(CFSItem newItem, CFSError error) { //YOUR CODE }];

  Move Folder

You can move a folder to a new location in the file system. If the destination conflicts with the moving folder you can either RENAME, OVERWRITE or FAIL the operation.

[folder moveToDestinationContainer:destinationFolder whenExists:CFSExitsRename completion:^(CFSItem movedItem, CFSError error) { //YOUR CODE }];

  Delete Folder

You can perform the delete operation on a folder. This will return the Success/fail status of the operation.

[folder deleteWithCommit:YES force:NO completion:^(BOOL success, CFSError *error) { //YOUR CODE }];

  Restore Folder

You can restore a Folder from the trash. The restore method can be set to either FAIL, RESCUE or RECREATE. This will return the Success/failure status of the operation.

[folder restoreToContainer:destinationFolder restoreMethod:RestoreOptionsFail restoreArgument:path completion:^(BOOL success, CFSError *error) { //YOUR CODE }];

  Create Sub Folder

You can create a sub folder in a specific folder. If the folder already has a sub folder with the given name, the operation will fail.

[folder createFolder:folderName whenExists:CFSItemExistsRename completion:^(CFSFolder newDir, CFSError error) { //YOUR CODE }];

  Upload File

You can upload a file from your local file system into a specific folder. If the destination conflicts, you can either RENAME, OVERWRITE or FAIL the operation.

[folder upload:localFilePath progress:^(NSInteger uploadId, NSString path, int64_t completedBytes, int64_t totalBytes) { } completion:^(NSInteger uploadId, NSString path, CFSFile cfsFile, CFSError error) { //your code }];

[share setPasswordTo:newPassword from:@oldPassword completion:^(BOOL success, CFSError *error) { //your code} ];
